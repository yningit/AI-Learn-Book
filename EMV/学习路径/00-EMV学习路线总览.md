# EMV 学习路线总览

这套文档把原来的大而全文档拆成了适合渐进阅读的学习路径，目标是做到：

- 从零开始也能建立 EMV 心智模型
- 从规范术语逐步过渡到交易控制与密码学
- 最后落到 POS Kernel 软件架构、调试方法和工程实践

## 文档定位

- [01-EMV基础入门](01-EMV基础入门.md)：建立 EMV 最基本的认知
- [02-交易流程与核心对象](02-交易流程与核心对象.md)：理解交易 12 步主线，以及 AID、AIP、AFL、TVR、TSI、CID 这些核心对象
- [03-密码学、联机与脚本](03-密码学、联机与脚本.md)：理解 ODA、ARQC/ARPC、Field 55、Issuer Script

- [06-真实交易日志逐行解读](06-真实交易日志逐行解读.md)：通过一笔教学用交易日志，学会把阶段、对象和决策串起来
- [07-品牌差异入门](07-品牌差异入门.md)：理解通用 EMV 骨架与品牌差异的边界，以及工程隔离方式

## 每册要解决的问题

| 文档 | 重点回答的问题 |
| --- | --- |
| 基础入门 | EMV 到底是什么？APDU、TLV、DOL 在干什么？接触与非接有什么区别？ |
| 交易流程与对象 | 一笔 EMV 交易按什么顺序跑？每一步产生哪些关键对象？ |
| 密码学与联机 | ODA、ARQC、ARPC、Field 55、Script 到底在解决什么安全问题？ |
| 工程实践 | 如果把 EMV 做成 POS Kernel，系统应该如何分层和建模？ |
| 调试与测试 | 为什么一笔交易失败？应该先看哪些日志和字段？ |
| 日志解读 | 如何把一份交易日志读成“阶段 + 对象 + 决策”的结构化结论？ |
| 品牌差异 | 哪些是 EMV 共性，哪些是 Visa / Mastercard / UnionPay 等品牌扩展？ |
| L2 测试 | Kernel L2 到底在测什么、常见失败模式是什么？ |
| 参数治理 | 为什么参数治理决定终端交易一致性和可维护性？ |
| 品牌对照 | 不同品牌的差异主要落在哪些工程模块和对象上？ |
| 真实案例 | 现场高频问题应该如何按现象、日志、对象、根因来分析？ |
| 卡组织全景 | 如何从参考书视角理解多卡组织差异的全貌与工程落点？ |
| 术语索引 | 某个 EMV 术语到底是什么意思、解决什么问题？ |
| Tag 索引 | 某个 Tag 常出现在哪个阶段、作用是什么？ |
| APDU 索引 | 某条 APDU 命令在交易里干什么、出问题时怎么查？ |
| 位级速查 | AIP、TSI、TVR 这些位图到底该怎么查、怎么联动看？ |
| AID/RID/CAPK | AID、RID、Index、CAPK 之间是什么关系，为什么 ODA 常要一起查？ |

## 建议学习方法

- 第一轮先看“每个章节在解决什么问题”
- 第二轮再关注 Tag、位图、APDU、报文细节
- 第三轮结合真实日志和测试 case 做回放
- 遇到不懂的对象，优先回到全量参考手册查细节，而不是试图一次记住所有 bit
