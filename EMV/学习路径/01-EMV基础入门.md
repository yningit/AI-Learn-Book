# 01 - EMV 基础入门

## 1. 先用一句话理解 EMV

EMV 是一套让芯片卡、终端和发卡行共同参与交易决策的规范体系，它的核心目标是：

- 识别卡上的支付应用
- 验证卡和持卡人是否可信
- 判断交易应离线通过、联机还是拒绝
- 在需要时把结果安全地同步给发卡行和卡片

## 2. EMV 不是只属于“卡片”

很多初学者会误以为 EMV 只是卡片规范，实际上它至少涉及 4 类角色：

1. 卡片：保存应用、密钥、计数器和风险状态
2. 终端：读取卡片、执行交易流程、做本地风险判断
3. 收单 / 网络：转发授权消息
4. 发卡行：做最终账户和风险决策

所以 EMV 不是单机系统，而是一个分布式交易系统。

## 3. 先建立 4 个核心问题

理解 EMV 时，先围绕下面 4 个问题思考：

1. 卡上有哪些应用可选？
2. 卡和终端各自支持什么能力？
3. 这笔交易应该走离线、联机还是拒绝？
4. 如果联机，卡片和发卡行如何完成安全确认？

后面的所有术语，其实都是在回答这 4 个问题。

## 4. 你必须先认识的三个基础：APDU、TLV、DOL

### 4.1 APDU

APDU 是终端和卡片之间的命令/响应协议。你可以简单理解成：

- 终端发命令给卡片
- 卡片回响应给终端

常见命令有：

- `SELECT`
- `GET PROCESSING OPTIONS`
- `READ RECORD`
- `VERIFY`
- `GENERATE AC`
- `EXTERNAL AUTHENTICATE`

### 4.2 TLV

TLV = Tag + Length + Value。

它是 EMV 中最常见的数据编码方式。你可以把它理解为一种二进制字段字典：

- Tag：字段编号
- Length：字段长度
- Value：字段值

### 4.3 DOL

DOL（Data Object List）不是字段值，而是“卡片要求终端提供哪些字段，以及每个字段长度是多少”的清单。

很多 GPO 或 Generate AC 的问题，最终都能追溯到 DOL 组装错误。

## 5. 接触式与非接式的差别

### 5.1 共同点

- 都遵循 EMV 交易骨架
- 都要做应用选择、读数据、风险判断和 AC 处理
- 都会使用 TLV 对象和 APDU 交互

### 5.2 差别

| 维度 | 接触式 | 非接式 |
| --- | --- | --- |
| 建链方式 | 上电、ATR、协议协商 | Polling、Anti-collision、ATS |
| 入口方式 | 通常直接进入接触 Kernel | 通常先走 PPSE 和 Entry Point |
| 时延要求 | 相对宽松 | 对响应速度更敏感 |
| 决策风格 | 更偏完整状态机 | 更常见统一 Outcome 模型 |

## 6. 初学者最先要记住的 EMV 对象

| 对象 | 作用 |
| --- | --- |
| AID | 标识支付应用 |
| AIP | 卡支持什么能力 |
| AFL | 终端接下来要读哪些记录 |
| TVR | 终端发现了哪些问题 |
| TSI | 终端做了哪些步骤 |
| CID | 卡对这笔交易给出的态度 |
| ATC | 卡片交易计数器 |
| 9F34 | 持卡人验证执行结果 |

## 7. 你应该如何理解一笔 EMV 交易

不要把 EMV 看成很多零散标签，而要把它看成一条连续决策链：

1. 选应用
2. 建立卡片能力图谱
3. 读应用数据
4. 做认证和持卡人验证
5. 做风险判断
6. 让卡片通过 Generate AC 给出态度
7. 如有需要，联机给发卡行做最终决策
8. 再把发卡行结果带回卡片完成闭环
