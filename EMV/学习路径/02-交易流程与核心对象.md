# 02 - 交易流程与核心对象

## 1. 为什么先学交易主线

EMV 最容易让人迷失的地方，是术语太多、标签太多、流程分支太多。

所以第二步不应该继续背术语，而应该先抓住一笔交易的主线。只要主线清楚，很多对象就知道该放在哪个阶段理解。

## 2. 一笔 EMV 交易的 12 步骨架

1. 交易初始化
2. 卡片激活 / 建链
3. Application Selection
4. GPO 前准备
5. GPO 返回 AIP / AFL
6. READ RECORD 读取应用数据
7. Offline Data Authentication
8. Processing Restrictions + CVM
9. Terminal Risk Management
10. Terminal Action Analysis
11. Generate AC / 联机 / 二次 Generate AC
12. 交易完成与后处理

## 3. 每一步在解决什么问题

| 步骤 | 它在解决什么问题 |
| --- | --- |
| 初始化 | 本次交易的金额、类型、终端能力是什么 |
| 激活 | 卡片链路是否建立成功 |
| 应用选择 | 这张卡上到底由哪个应用接管交易 |
| GPO 前准备 | 卡片要求终端先提供哪些数据 |
| GPO | 卡支持什么、终端接下来读什么 |
| READ RECORD | 把卡上的核心交易对象读出来 |
| ODA | 卡片数据是否可信 |
| CVM | 持卡人是否通过验证 |
| Risk | 是否应该更倾向联机 |
| Action Analysis | 基于 TVR / TAC / IAC 做动作决策 |
| Generate AC | 让卡片正式给出态度 |
| 完成交易 | 把交易结果、日志、脚本和持久化收口 |

## 4. 关键对象应该在哪个阶段理解

### 4.1 AID

- 出现在 Application Selection 阶段
- 决定后面由哪个应用或哪个品牌逻辑接管交易

### 4.2 AIP

- 出现在 GPO 响应中
- 告诉终端：卡支持哪些能力

### 4.3 AFL

- 同样出现在 GPO 响应中
- 告诉终端：下一步该读哪些记录

### 4.4 CVM List

- 通常在 READ RECORD 后拿到
- 告诉终端：这张卡希望按什么顺序做持卡人验证

### 4.5 TVR

- 在 ODA、Processing Restrictions、CVM、Risk、Script 等阶段逐步形成
- 它不是结论，而是事实记录

### 4.6 TSI

- 记录本次交易哪些关键步骤已经执行
- 适合和 TVR 联合阅读

### 4.7 CID

- 出现在 Generate AC 响应中
- 是卡片对交易表态的关键结果字段

## 5. GPO 是为什么这么重要

很多初学者会过度关注 `GENERATE AC`，但工程上 GPO 同样是非常关键的一步。

因为 GPO 同时回答两件事：

1. 卡支持什么能力
2. 终端还要去读哪些数据

如果 GPO 阶段组包错误，后面很多问题都只是连锁反应。

## 6. TVR、TSI、TAC、IAC 的关系

这是初学者常见的难点，可以先这样记：

- `TVR`：终端发现了什么问题
- `TSI`：终端做了哪些步骤
- `TAC`：终端对这些问题的动作策略
- `IAC`：发卡行对这些问题的动作策略

其中：

- TVR 是事实
- TAC/IAC 是规则
- Action Analysis 是把事实送进规则里，得到联机、拒绝或默认路径

## 7. Generate AC 为什么是交易核心

因为从这一步开始，卡片不再只是提供数据，而是开始正式参与交易决策。

卡片可能给出：

- AAC：拒绝
- TC：认可交易完成
- ARQC：要求联机

这也是为什么 `GENERATE AC` 经常被看作 EMV 交易的分水岭。

## 8. 一个建议的理解框架

把交易拆成 4 段最容易理解：

### 第一段：建立交易上下文

- 初始化
- 激活
- 应用选择

### 第二段：建立能力和数据画像

- GPO
- READ RECORD

### 第三段：做本地判断

- ODA
- CVM
- Risk
- Action Analysis

### 第四段：做最终决策闭环

- Generate AC
- 联机授权
- 二次 Generate AC
- 完成交易

## 9. 学这一册时最容易犯的错误

1. 记住了很多 Tag，但不知道 Tag 在哪个阶段出现
2. 知道 TVR 很重要，但不知道它是怎么一点点被置位的
3. 知道 ARQC、TC、AAC，但不知道它们是 Generate AC 的不同结果
4. 把 GPO 看成简单准备动作，而不是关键分叉点

## 10. 这一册学完后，你应该达到什么程度

读完后，你应该能：

- 用自己的话把 12 步流程讲清楚
- 说出 AID、AIP、AFL、TVR、TSI、CID 的阶段位置和作用
- 理解为什么很多问题必须回到 GPO、TVR 和 Generate AC 去看
