# 05 - 调试、排错与测试

## 1. 为什么很多人学了 EMV 还是不会排错

因为 EMV 的问题通常不会直接告诉你“是哪里错了”。

你看到的可能只是：

- GPO 失败
- ODA 失败
- 主机说 ARQC 校验失败
- 联机批准但卡片最终拒绝
- 非接频繁提示插卡

要真正定位问题，必须把日志、对象、状态机和交易阶段串起来看。

## 2. 读一笔交易日志的推荐顺序

1. 看接口类型：接触还是非接
2. 看应用选择：最终 AID、Kernel ID、Label 是否正确
3. 看 GPO：PDOL、AIP、AFL 是否合理
4. 看读记录：关键标签是否读全
5. 看 ODA / CVM / Risk：TSI 是否表明执行过，TVR 是否表明失败或命中规则
6. 看第一次 Generate AC：CID 是 AAC、TC 还是 ARQC
7. 看联机：Field 55 是否正确，主机响应是否完整
8. 看脚本与二次 AC：ARPC、Script、最终 CID 是否一致

## 3. 最小观察集

如果时间有限，优先抓下面这些对象：

| 类别 | 必看对象 |
| --- | --- |
| 应用选择 | AID、Label、Interface、Kernel ID |
| GPO | PDOL 输入、AIP、AFL |
| 读记录 | PAN、有效期、CVM List、CDOL1、CDOL2 |
| 决策 | TVR、TSI、9F34、TAC/IAC 命中情况 |
| AC | CID、ATC、AC、IAD |
| 联机 | Field 55、ARC、ARPC、Script |

## 4. 高频问题与优先排查项

### 4.1 SELECT 成功但 GPO 失败

优先排查：

- PDOL 是否组装正确
- 长度是否匹配
- TTQ / 金额 / 日期等是否正确填充

### 4.2 ODA 失败

优先排查：

- AIP 是否支持相应模式
- AFL 是否把认证相关记录读全
- CAPK 是否命中
- 证书链恢复与哈希输入是否正确

### 4.3 主机报 ARQC 校验失败

优先排查：

- `9F26`、`9F36`、`9F37`、`95`、`9F10`
- Field 55 编码与顺序
- UN 是否符合要求
- 标签值来源是否正确

### 4.4 主机批准但卡片最终拒绝

优先排查：

- ARPC 是否正确传回
- Issuer Script 是否执行失败
- 二次 Generate AC 输入是否完整
- TVR 中是否有 issuer/script 相关异常

### 4.5 非接经常要求插卡

优先排查：

- PPSE 是否返回候选应用
- Entry Point 是否正确路由
- TTQ、Reader capability、限额参数是否合理

## 5. 一套实用的排错模板

建议每次问题分析都按下面结构写：

### 问题现象

例如：主机批准，但终端显示失败。

### 交易阶段定位

- 第一次 AC 返回什么
- 联机返回什么
- 二次 AC 返回什么

### 关键对象

- AIP
- TVR
- TSI
- 9F34
- CID
- ARC
- ARPC
- Script 结果

### 初步结论

说明问题更像出在：

- 交易输入
- 风险决策
- 联机映射
- 主机回包消费
- 脚本或二次 AC

## 6. 测试时应该优先覆盖什么

### 6.1 流程类

- 正常消费
- 脱机批准
- 脱机拒绝
- 联机批准
- 联机拒绝
- 联机超时 / 通讯失败

### 6.2 安全类

- SDA / DDA / CDA 成功与失败
- Offline PIN / Online PIN / No CVM
- ARQC / ARPC 成功与失败
- Script 执行成功与失败

### 6.3 参数类

- Floor Limit 命中
- 随机联机命中
- TAC / IAC 不同组合
- CAPK / AID / Reader 参数版本切换

### 6.4 异常类

- 掉电恢复
- 卡移除
- 用户取消
- PIN 超时
- 非接场强波动

## 7. 日志系统应该长什么样

建议至少包含 5 类日志：

1. 状态迁移日志
2. APDU 日志
3. 标签快照日志
4. 决策日志
5. 联机映射日志

如果缺其中任何一类，现场定位效率都会显著下降。

## 8. 这一册学完后，你应该达到什么程度

读完后，你应该能：

- 知道遇到问题时该先看哪一段日志
- 能把问题落到交易阶段、关键对象和决策来源上
- 知道测试不只是跑通交易，还要覆盖风险、脚本、联机和异常恢复
