# 13 - 术语索引

这份术语索引用于把整套 EMV 文档里反复出现的核心术语集中整理，方便作为“参考书附录”快速查阅。

> 说明：术语解释以工程理解和交易语义为主，不替代正式规范定义。

## A

### AAC

Application Authentication Cryptogram。卡片在 `GENERATE AC` 中返回的一种结果类型，表示拒绝或不接受当前交易完成。

### AC

Application Cryptogram。交易过程中由卡片生成的密码学结果，常见类型包括 AAC、TC、ARQC。

### ADF

Application Dedicated File。卡上的应用文件或应用目录实体，终端通常通过 `SELECT` 进入。

### AFL

Application File Locator。卡片在 GPO 后返回的“读记录计划表”，告诉终端后续需要读哪些记录。

### AID

Application Identifier。支付应用标识符，是应用选择阶段最核心的对象之一。

### AIP

Application Interchange Profile。卡片能力位图，告诉终端卡支持哪些 EMV 能力，例如 ODA、CVM、Issuer Authentication、CDA。

### Amex

American Express，美国运通。常见国际卡组织之一，具有自身的品牌规则和测试要求。

### ARC

Authorization Response Code。主机授权响应码，表示联机授权结果，如批准、拒绝、需进一步处理等。

### ARPC

Authorization Response Cryptogram。发卡行对 ARQC 的回应，用于让卡片确认联机响应来自可信发卡方。

### ARQC

Authorization Request Cryptogram。卡片生成的联机请求密码，用于让发卡行验证交易合法性和关键数据一致性。

### ATR

Answer To Reset。接触式卡片上电复位后返回的初始响应。

### ATS

Answer To Select。非接式卡片在激活过程中返回的响应数据，类似非接链路的能力协商结果。

## C

### CAPK

Certification Authority Public Key。终端预置的 CA 公钥，是 ODA 证书链的信任根。

### Cardholder Verification Method（CVM）

持卡人验证方式，用于确认持卡人身份，例如 Offline PIN、Online PIN、Signature、No CVM。

### CDA

Combined DDA / Application Cryptogram。把动态认证与应用密码结合在一起的离线认证方式。

### CID

Cryptogram Information Data。`GENERATE AC` 返回中的关键结果字节，用于指示 AAC、TC、ARQC 等类型。

### CVM Limit

非接小额交易中常见的阈值参数，用于决定是否需要进行 CVM。

### CVM List

卡片提供的持卡人验证规则清单，规定不同条件下优先尝试哪些 CVM。

### CVR

Card Verification Results。常见于品牌私有 IAD 结构中的风险状态数据，不同卡组织解释方式通常不同。

## D

### DDA

Dynamic Data Authentication。动态数据认证，可证明卡片具备对应私钥能力，安全性高于 SDA。

### DF Name

Dedicated File Name。常见 Tag `84`，通常用于表示应用名称或目录名称。

### Diners Club

大莱卡组织，常与 Discover 网络和受理体系一起讨论。

### Discover

Discover 卡组织。工程实现上常与 Diners/受理网络兼容性一起考虑。

### DOL

Data Object List。数据对象清单，用来规定终端应按什么顺序和长度提供数据，而不是字段值本身。

## E

### EMV

源自 Europay、Mastercard、Visa。现在通常指芯片支付交易规范体系，而不再只指三家机构本身。

### Entry Point

非接触交易中的入口路由层，用于在多品牌、多 Kernel 间做候选应用发现和路由决策。

### Exception File

终端本地异常名单或黑名单，用于风险控制。

## F

### Field 55

ISO8583 中承载 EMV TLV 的关键字段，通常用于上传芯片交易关键对象。

### Floor Limit

终端本地金额阈值。超过该阈值的交易通常更倾向联机。

## G

### GPO

GET PROCESSING OPTIONS。GPO 是从应用选择进入交易执行阶段的关键命令，通常返回 AIP 和 AFL。

## H

### Host Mapping

把终端内核上下文映射为主机字段和报文的过程，常按品牌隔离实现。

## I

### IAC

Issuer Action Codes。发卡行在卡片中提供的动作码集合，用于与 TVR 联合决定联机、拒绝或默认行为。

### IAD

Issuer Application Data。发卡行定义的应用数据，常用于联机风控和品牌私有解释。

### ICC

Integrated Circuit Card。芯片卡。

### Issuer Script

发卡行在联机响应中下发给卡片的远程命令，常见标签模板为 `71` 和 `72`。

## J

### JCB

日本国际卡组织之一，具有独立品牌规则、参数和认证要求。

## K

### Kernel

终端中实现 EMV 交易流程的核心协议引擎。

### Kernel ID

非接触场景下常见的品牌或内核标识，用于 Entry Point 路由。

## L

### L1

通常指底层物理链路和读卡传输能力层。

### L2

通常指 EMV Kernel 行为实现和品牌规则执行层。

### L3

通常指终端业务与主机集成层的测试和交付层面。

## O

### ODA

Offline Data Authentication。离线数据认证，用于验证卡片静态或动态数据真实性，包含 SDA、DDA、CDA。

### Outcome

非接触场景中常见的统一结果模型，用于对上层输出最终处理建议和用户提示。

## P

### PAN

Primary Account Number。主账号或卡号。

### PDOL

Processing Options Data Object List。卡片在 GPO 前告诉终端需要提供哪些数据对象。

### PICC

Proximity Integrated Circuit Card。非接式芯片卡。

### PIN Bypass

终端支持 PIN，但用户选择跳过 PIN 输入的情形，需谨慎处理和记录。

### PPSE

Proximity Payment System Environment。非接场景常见目录名，用于发现候选支付应用。

### Pure

某些地区或方案下会使用的本地或区域支付 profile / scheme，工程上应作为独立卡组织 profile 处理。

## R

### Reader Limit

非接终端参数，用于限制非接受理能力、金额范围或路径选择。

### RID

Registered Application Provider Identifier。AID 前 5 字节中的注册应用提供方标识，常用于命中 CAPK 集合。

### RuPay

印度本地卡组织之一，工程上常需要结合本地化参数、主机和受理规则来实现。

## S

### SDA

Static Data Authentication。静态数据认证，主要验证静态数据完整性，但防复制能力弱于 DDA / CDA。

### Scheme Profile

工程上为不同卡组织抽象的配置、规则、对象解释和 host mapping 集合。

### Script 71 / 72

Issuer Script 的常见模板标签。工程上通常理解为最终 AC 前 / 后执行的脚本模板。

### SELECT

EMV 中用于选择目录或应用的 APDU 命令。

### SmartPOS

集成了支付与业务能力的智能终端形态，常内置 EMV Kernel。

### SoftPOS

基于通用设备实现的受理方案，可能涉及非接内核、设备能力约束和软件安全要求。

## T

### TAC

Terminal Action Codes。终端侧动作码集合，用于与 TVR 联合做决策。

### TC

Transaction Certificate。卡片在 `GENERATE AC` 中返回的一种结果类型，通常表示卡片接受交易完成。

### TLV

Tag-Length-Value。EMV 中最常见的数据编码结构。

### TMS

Terminal Management System。终端参数、应用、证书或配置的下载和管理平台。

### TSI

Transaction Status Information。记录本次交易执行过哪些关键步骤。

### TTQ

Terminal Transaction Qualifiers。非接场景常见终端能力 / 交易能力对象。

### TVR

Terminal Verification Results。终端在交易中发现的问题事实集，是动作分析的重要输入。

## U

### UN

Unpredictable Number。终端生成的不可预测数，参与 ARQC 等关键计算。

### UnionPay

银联卡组织，工程上常与本地化受理场景、参数和主机协作一起考虑。

## V

### Velocity Checking

风险管理的一部分，用于检查交易频率、累计金额或相关计数器。

### Visa

国际卡组织之一，也是很多 EMV 通用资料中的高频参考品牌。
