# 16 - 位级速查附录

这份附录把 EMV 中最常被问到、也最常用于排错的几个关键位图集中整理，方便在日志分析、联调和测试时快速查阅。

重点覆盖：

- AIP
- TSI
- TVR

> 说明：位级语义在不同规范版本和品牌实现中可能有补充差异；本附录优先给出通用、高频、工程最常用的理解方式。

## 一、使用这份附录的正确方法

不要把这份附录当成“死背 bit 表”，而应按下面顺序使用：

1. 先判断交易走到哪个阶段
2. 再看对应位图记录了什么
3. 最后结合其他对象决定问题是“没做”“做了但失败”“规则命中”还是“品牌差异”

## 二、AIP 速查

AIP（Application Interchange Profile）是卡片在 GPO 后返回的能力位图。

### 1. AIP 最重要的工程价值

AIP 不只是告诉终端“卡支持什么”，更重要的是：

- 它决定后续哪些流程应该执行
- 它决定哪些失败属于“没实现”还是“本来就不支持”
- 它决定联机后是否需要 issuer authentication 等路径

### 2. AIP 高频位速查

| 常见位 | 常见语义 | 工程排查意义 |
| --- | --- | --- |
| Byte1 b8 | SDA supported | 可进入 SDA 路径 |
| Byte1 b7 | DDA supported | 可进入 DDA 路径 |
| Byte1 b6 | Cardholder verification supported | 终端才有正常 CVM 前提 |
| Byte1 b5 | Terminal risk management to be performed | 不应跳过风险管理 |
| Byte1 b4 | Issuer authentication supported | 联机后可能需 issuer auth / ARPC 路径 |
| Byte1 b2 | CDA supported | `GENERATE AC` 时可能需联合动态认证 |

### 3. AIP 常见排错问题

#### 现象：ODA 没做

优先判断：

- AIP 是否真的支持 SDA / DDA / CDA
- 终端是否正确读取了 AIP
- 终端是否因为参数或品牌 profile 误跳过流程

#### 现象：CVM 行为怪异

优先判断：

- AIP 是否支持持卡人验证
- 若卡本身不支持 CVM，就不要把问题先归到 CVM List 或 PIN Pad

## 三、TSI 速查

TSI（Transaction Status Information）用于记录：**本次交易到底做过哪些步骤。**

### 1. TSI 的核心价值

TSI 最适合回答的问题是：

- 这个步骤到底有没有执行过？
- 日志里写“做了”，系统状态里是否也真的记录成“做了”？

### 2. TSI 高频项速查

| 高频项 | 常见理解 | 工程排查意义 |
| --- | --- | --- |
| Offline data authentication was performed | 做过 ODA | 若 TVR 显示 ODA failed，说明“做了但失败” |
| Cardholder verification was performed | 做过 CVM | 若 `9F34` 异常，说明 CVM 结果或映射有问题 |
| Card risk management was performed | 做过风险管理 | 若联机路径异常，要看是否真做过风险判断 |
| Issuer authentication was performed | 做过发卡行认证 | 主机批准后卡片异常时要结合此项看 |
| Script processing was performed | 做过脚本处理 | 若 TVR 有 script failed，说明不是“没做”，而是“做失败了” |

### 3. TSI 排错口诀

- `TSI = 做没做`
- `TVR = 做的时候发现了什么问题`

## 四、TVR 速查

TVR（Terminal Verification Results）是终端事实记录器。它不是最终动作结论，而是给 TAC/IAC 决策提供输入。

### 1. 不建议直接死背 5 字节

更有效的理解方法是按阶段分组：

- ODA 相关
- Processing Restrictions 相关
- CVM 相关
- Risk 相关
- Issuer / Script 相关

### 2. ODA 相关高频位

常见语义包括：

- Offline data authentication was not performed
- SDA failed
- DDA failed
- CDA failed
- ICC data missing for offline authentication

**工程排查重点：**

- AIP 是否支持
- AFL / READ RECORD 是否读全认证对象
- CAPK / RID / Index 是否匹配
- 哈希、证书链、模板解析是否正确

### 3. Processing Restrictions 相关高频位

常见语义包括：

- 应用版本不匹配
- 卡已过期
- 卡尚未生效
- 使用控制条件不满足

**工程排查重点：**

- 日期、国家码、交易类型输入是否正确
- 卡片有效期对象是否正确解析
- 参数是否与品牌 / 终端场景匹配

### 4. CVM 相关高频位

常见语义包括：

- Cardholder verification was not successful
- Unrecognised CVM
- PIN try limit exceeded
- PIN entry required and pad not present

**工程排查重点：**

- AIP 是否支持 CVM
- CVM List 解释是否正确
- 终端 / PIN Pad 能力是否匹配
- `9F34` 是否和 UI 行为一致

### 5. Risk 相关高频位

常见语义包括：

- Transaction exceeds floor limit
- Random transaction selected for online processing
- Merchant forced transaction online

**工程排查重点：**

- Floor Limit 是否配置正确
- 随机联机参数是否合理
- 是否为业务强制联机，而不是技术失败

### 6. Issuer / Script 相关高频位

常见语义包括：

- Issuer authentication failed
- Script processing failed before final `GENERATE AC`
- Script processing failed after final `GENERATE AC`

**工程排查重点：**

- ARPC 是否回传并正确消费
- 71/72 脚本时机是否正确
- 二次 Generate AC 输入是否完整

## 五、联合分析模板

实际排错时，建议固定按下面组合看：

| 组合 | 用途 |
| --- | --- |
| AIP + TSI | 卡支持什么，终端是否真的执行了 |
| TSI + TVR | 区分“没做”还是“做了但失败” |
| TVR + TAC/IAC | 判断联机 / 拒绝 / 默认路径来源 |
| TVR + `9F34` + CID | 判断 CVM、风险与最终 AC 结果关系 |

## 六、一个快速判断表

| 现象 | 第一优先位图 | 第二优先对象 |
| --- | --- | --- |
| ODA 异常 | AIP、TSI、TVR | CAPK / ODA 对象 |
| CVM 异常 | AIP、TSI、TVR | `8E`、`9F34` |
| 联机倾向异常 | TVR | TAC / IAC |
| 主机批准但卡拒绝 | TSI、TVR | `8A`、`91`、Script、CID |

## 七、初学者最容易误读的地方

1. 把 TVR 当最终结论，而不是事实集。  
2. 看到 AIP 支持，就以为终端一定已经正确执行。  
3. 只看 TVR，不看 TSI。  
4. 忽略品牌对 bit 使用和解释的差异。
