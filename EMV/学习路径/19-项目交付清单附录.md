# 19 - 项目交付清单附录

这份附录面向项目上线、版本封版、验收和交付准备，适合作为终端项目的最后检查单。

它的目标不是替代研发流程，而是确保 EMV 项目在上线前至少覆盖最关键的准备项。

## 一、版本与发布准备

### 1. 代码版本

- 是否有明确版本号
- 是否有封版记录
- 是否确认本次版本包含哪些功能 / 修复
- 是否有已知限制说明

### 2. 发布包

- 是否有正式发布包校验方式
- 是否区分测试 / 预发 / 生产包
- 是否保留回滚版本

## 二、参数与配置准备

### 1. AID 参数

- AID 集合是否完整
- 优先级和部分匹配策略是否确认
- 接触 / 非接 profile 是否匹配

### 2. CAPK 与证书

- CAPK 是否完整
- CAPK 是否区分测试 / 生产环境
- CAPK 版本和有效期是否确认

### 3. 风险参数

- TAC 是否确认
- Floor Limit 是否确认
- Reader / CVM / Contactless limit 是否确认

### 4. 参数治理能力

- 是否能确认终端当前参数版本
- 是否支持参数回滚
- 是否保留参数发布记录

## 三、交易功能准备

### 1. 基本交易路径

- 正常联机批准
- 联机拒绝
- 脱机批准 / 拒绝（如适用）
- 非接正常受理
- 接触回退（如适用）

### 2. 持卡人验证路径

- Offline PIN
- Online PIN
- Signature
- No CVM
- PIN bypass / timeout / cancel

### 3. 联机后处理路径

- ARPC 返回和消费
- Issuer Script 执行
- 二次 Generate AC

## 四、异常与恢复准备

### 1. 超时场景

- 联机超时后是否正确收口
- UI 是否提示明确
- 是否触发冲正或待确认状态

### 2. 中断场景

- 掉电恢复
- 卡片中途拔出
- 用户取消
- 打印失败

### 3. 结果一致性

- 内核结果、业务结果、主机结果是否可对齐
- 是否能通过日志回放定位异常

## 五、日志与可观测性准备

### 1. 必备日志

- 状态迁移日志
- APDU 日志
- TVR / TSI / 9F34 / CID 快照
- 联机前 Field 55 快照
- 主机响应与 Script 摘要

### 2. 版本与参数可观测性

- 日志是否打印代码版本
- 日志是否打印参数版本
- 日志是否打印命中的 AID profile / CAPK 标识

## 六、主机联调准备

### 1. 报文确认

- ISO8583 字段映射是否确认
- Field 55 标签集合是否确认
- 响应码、授权码、脚本、ARPC 字段是否确认

### 2. 错误码确认

- 主机失败码是否有说明
- ARQC verify fail、script fail、timeout 等场景是否有统一解释

### 3. 环境确认

- 测试环境与生产环境是否区分
- 主机地址、证书、路由、白名单是否确认

## 七、测试准备

### 1. 功能测试

- 接触 / 非接
- 成功 / 失败 / 超时
- 不同 CVM 路径

### 2. L2 / 品牌测试

- 重点 case 是否通过
- 已知不支持或限制项是否明确

### 3. 回归测试

- 当前版本变更影响范围是否覆盖
- 历史高风险路径是否回归

## 八、交付资料准备

上线或交付前，建议至少准备：

- 版本说明
- 功能范围说明
- 参数版本说明
- 已通过测试清单
- 已知问题清单
- 回滚方案
- 现场支持联系人清单

## 十、最容易遗漏的 10 个点

1. 忘记确认参数版本
2. CAPK 测试 / 生产混用
3. 只测成功交易，没测超时和冲正
4. 日志没带版本和参数信息
5. Host mapping 文档不完整
6. 非接参数未按品牌确认
7. Script 成功路径测了，失败路径没测
8. 二次 Generate AC 没独立观察
9. 终端 UI 结果和内核结果未对齐验证
10. 没有明确回滚包和回滚流程

## 十一、建议的交付门槛

如果下面任一项不满足，建议不要轻易进入正式上线：

- 关键交易路径未完整验证
- 参数版本不可追踪
- 主机联调失败原因不明确
- 异常恢复和冲正策略未验证
- 无法快速回滚
