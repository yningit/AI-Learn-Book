# 全书总纲与章节路线

## 1. 这份总纲的定位

随着文档集逐步扩展，内容已经从一篇入门说明，发展成一套覆盖：

- 体系理解
- 设备安全
- 密钥机制
- 终端注入
- HSM 控制
- 报文安全
- PIN 域
- MAC 完整性
- 密钥运输层
- 令牌化与去敏
- 生命周期规范

的完整学习材料。

这份总纲的目的，是把整套内容重新组织成一本“书”的阅读路线，而不是一组松散文档。

## 2. 这套书究竟在讲什么

如果把整套文档压缩成一句话，它讲的是：

- 在支付终端到主机的链路里，敏感对象如何被采集、加密、认证、传输、解密、验证、运维和审计。

也就是说，这不是单纯一本讲 DUKPT 的资料，也不是单纯一本讲 P2PE 的资料，而是一套围绕支付终端到主机、从生产到运维的数据安全体系教材。

## 4. 三条贯穿全书的主线

### 主线一：对象主线

关注：

- PAN
- Track Data
- PIN / PIN Block
- KSN
- 密文载荷
- MAC

这条线回答的是：

- 到底保护的是什么对象。

### 主线二：边界主线

关注：

- 终端边界
- 设备读取边界
- 商户边界
- 网关边界
- 解密域边界
- HSM 边界

这条线回答的是：

- 什么东西应该出现在哪里。

### 主线三：密钥主线

关注：

- BDK / IPEK / 工作键
- MK / SK / Session Key
- PIN / Data / MAC 用途键
- LMK / ZMK / TMK
- Key Block / TR-31

这条线回答的是：

- 密钥如何分层、分域、分用途、分运输。

### 主线四：数据治理主线

关注：

- Token
- 脱敏
- Detokenization
- 日志治理
- 数据保留

这条线回答的是：

- 解密之后如何继续缩小暴露面。

## 5. 如果你只想抓住全书的骨架

可以把整套文档压缩成下面 10 个问题：

1. P2PE 在缩小什么暴露面。
2. DUKPT 在解决什么密钥问题。
3. SRED 在定义什么样的终端读取起点。
4. MK/SK 为什么仍然是很多 POS 系统的重要主线。
5. PIN / Data / MAC 为什么必须分开看。
6. HSM 为什么不只是一个密码计算盒子。
7. 终端注入为什么本质上是安全运营流程。
8. 报文为什么既要加密，又要做完整性认证。
9. 为什么解密后还要继续做 Tokenization 和数据最小化。
10. 为什么真实项目的难点常常不在算法，而在状态、台账、用途、供应链和边界。

## 6. 一句总结

这套文档最适合的阅读方式，不是随机挑一篇看完就结束，而是沿着“对象、边界、密钥、数据治理”四条主线反复回看。只要这几条线在你脑中合成一张图，POS 数据安全这块知识就会真正成体系。
