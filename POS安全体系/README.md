# POS 数据安全体系文档集

这是一套面向支付安全学习与项目入门的 Markdown 文档集。它最初围绕 P2PE / DUKPT 展开，现在已经扩展为覆盖 POS 终端到主机全链路的数据安全体系，目标不是堆术语，而是帮助你把“支付链路、安全边界、设备安全、密钥体系、工程实现、合规运营、数据治理”连成一张完整的脑图。

## 文件夹结构

- `README.md`：项目入口，保留在根目录。
- `00_导航总览/`：参考书、学习地图、全书总纲等导航类文档。
- `01_体系总览/`：POS 安全全景与总览扩展文档。
- `02_终端与链路保护/`：P2PE、DUKPT、SRED、PIN、MAC 等终端采集与链路保护主题。
- `03_密钥体系与HSM/`：用途键、MK/SK、HSM、注入、TR-31 等密钥治理主题。
- `04_报文对象与生命周期/`：交易流程、字段安全、支付对象、生命周期、伪报文示例。
- `05_数据治理与合规/`：实施合规、Tokenization、生命周期规范矩阵。
- `06_评审排障与案例/`：架构评审、排障演练等落地类文档。
- `07_厂商语境与协议附录/`：厂商命名语境和协议适配附录。

这样分层后，后续继续扩展 POS 生产、终端管理、主机安全、软件合规时，都可以按主题落到对应目录，而不需要继续堆在根目录。

## 建议如何使用这套文档

- 如果你是第一次接触，先把总览文档完整看完，再进入专题篇。
- 如果你是开发或架构人员，建议优先看 DUKPT 深入篇和实施指南。
- 如果你是做方案沟通或培训，图解篇会更适合做内部材料基础。
- 如果你后续要做代码或报文实现，这套文档更适合作为“概念和架构底稿”，而不是直接替代规范。
- 如果你要做实现、联调或排障，建议继续阅读算法篇、字段安全篇和排障手册。
- 如果你想把所有内容真正串成一体，建议最后阅读数据对象篇、报文生命周期篇和学习地图。
- 如果你开始接触终端注入、HSM key slot 或用途键配置，建议继续看密钥分区专题。
